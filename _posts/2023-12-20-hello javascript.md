### 프로그래밍 언어들이 다양한 이유

- **컴파일 여부**  
  컴파일 : 프로그래밍 언어로 작성된 코드를 컴퓨터가 이해할 수 있도록 0과 1로 표현된 기계어로 변환하는 과정  
  컴파일 언어(C언어, Java 등) : 소스코드가 실행되기 전에 코드를 기계어로 바꿔두기 때문에 처음 실행시 컴파일 하는 시간이 필요하다. 그 과정에서 에러를 미리 마주하여 런타임에 에러를 마주할 확률이 적어진다.  
  인터프리터 언어(Python, Javascript) : 실행과 동시에 한 줄 한 줄 해석하고 실행한다.

- **성능/안정성과 편의성의 Trade-off 관계**  
  Trade off : 하나를 얻으면 하나를 잃는 관계  
  프로그램이 동작하기 위해선 메모리와 연산능력이 필요하다. 프로그래밍 언어 자체적으로 많은 것들을 지원해줄수록(메모리 관리, 타입 추론 등) 해당 언어로 작성된 프로그램의 성능과 때론 안정성이 일반적으로 떨어지게 된다.

- **패러다임**  
  패러다임 : 특정 프로그램을 작성할 때 설계단계와 개발, 문제 해결을 수행하는 데에 있어 주요하게 작용하는 특정한 방법론과 사고의 흐름  
   객체지향형, 절차형, 함수형이 있으며, 프로그래밍 언어들은 특정 패러다임에 관련한 문법을 지원하거나 지원하지 않기 때문에 언어는 다양해질 수 밖에 없다. 자세한 건 프로그래밍 언어 표준, ECMA를 공부하면 된다.

### 엄격 모드

2009년 새롭게 제정된 ES5에서는 기존 기능 일부가 변경되어 하위 호환성 문제가 생길 수 있게 되었다. 그래서 대부분의 변경사항을 ES5의 기본 모드에선 활성화되지 않도록 설계해, `**"use strict";**` 라는 특별한 지시자로 엄격 모드를 활성화 했을 때만 변경사항이 활성화 되도록 한다. 기본 모드에선 무시되던 에러들이 뜰 수 있다.

`"use strict";` 가 스크립트 최상단에 오면 그 스크립트 전체를, 함수 본문 최상단에 오면 해당 함수만 엄격 모드로 실행한다. 무조건 **최상단에** 와야 하며, **엄격 모드를 취소하는 방법은 없다.** 브라우저 콘솔을 사용하는 경우엔 기본적으로 sloppy mode(non-strict)이며, 브라우저에서 테스트를 해야 할 때엔 이렇게 하면 된다.

```javascript
"use strict"; <Shift + Enter를 눌러 줄바꿈>
// ... 테스트 코드 입력
<Enter를 눌러 실행>
```

모던 자바스크립트는 클래스와 모듈이라 불리는 진일보한 구조를 제공하는데 이 둘을 사용하면 엄격 모드가 자동 적용되어 수동으로 선언하지 않아도 된다.

### 변수? 표현식? 문?

**문(statement)** : 값을 가지지 않음, 프로그램 상태를 변경(변수에 값 할당, 함수 도출)

**표현식(expression)** : 값을 가짐 : 상태를 변경하지 않음

```javascript
10 + 20;
```

위와 같은 코드를 계산할 때, 10, 20, +와 같은 기호(리터럴과 연산자)를 저장하는 메모리와 이 연산을 수행하는 연산능력 CPU로 하여금 연산과 기억이 나뉘어 수행된다. **메모리**는 용량을 기준으로 나뉘어져 있고, 데이터를 저장할 수 있는 각각의 메모리 셀의 집약체이다. 그 칸마다 고유한 메모리의 주소가 붙어져있고, 여기에 우리가 알고 있는 0과 1로 표현된 데이터가 올라간다. 필요에 따라 해당 칸의 주소로 접근하게 된다.

위 코드 실행 시 30이 나오겠지만 이는 저장되지 않으며 나온값 30을 다시 활용할 수도 없다. 메모리 셀의 주소를 하나하나 찾을 수 없을 뿐더러 프로그램 실행마다 주소는 변경되기 때문이다. 그래서 이를 재사용하기 위해 **변수**라는 메커니즘을 이용한다.

1. 변수 영역에서 빈공간을 확보해 식별자를 저장하고,
2. 데이터 영역의 변수 영역에서 빈공간을 확보하여 데이터(주소)를 저장한다.
3. 변수 영역에서 해당 식별자로 검색하여 데이터 영역에 저장한 데이터의 주소를 변수 영역의 값에 할당한다.

### 변수와 상수

_자바스크립트는 **동적 타입 언어**이며 우리가 타입을 지정해 줄 필요는 없지만 그렇다고 타입이 없는 건 아니며, 특정한 규칙이 존재한다._

- **변수**

  ```javascript
  let message;
  message = "Hello!";
  alert(message);
  ```

  `**let**` 키워드로 변수를 선언한다. 자료형은 뭐든 될 수 있다. 이의 오래된 버전은 `var` 이며 `let` 과 동일하게 작용하지만 미묘한 차이가 있다. ([참고](https://ko.javascript.info/var)) `**alert**` 는 경고창(대화창)을 띄워 출력하는 키워드이다.  
  엄격 모드를 사용하지 않을 땐 `let` 없이 `num = 5;` 만 써 단순 할당하여도 변수가 생성된다. 하지만 엄격 모드에선 에러가 나기 때문에 굳이 저렇게 쓸 필요는 없다.

- **상수**  
  변화하지 않는 변수를 선언할 땐 `let` 대신 `**const**` 를 사용한다. 재할당을 할 수 없는 상수 키워드이다. 대문자 상수는 하드 코딩한 값의 별칭을 만들 때 사용한다. e.g. rgb

- **변수 명명 규칙**

  - 오직 문자, 숫자, 기호 $, 기호 \_ 만 들어갈 수 있다.
  - 첫 글자는 숫자가 될 수 없다.
  - camelCase
  - 대소문자를 구분한다. (apple ≠ AppLe)
  - 예약어는 사용할 수 없다.

- **바람직한 변수명 짓기**
  - `userName` 이나 `shoppingCart` 처럼 읽을 수 있는 이름을 사용
  - 동작을 명확히 알지 못하는 경우 외에는 줄임말이나 `a`, `b`, `c` 와 같은 짧은 이름은 지양
  - 최대한 서술적이고 간결하게 명명한다.  
    `data` 와 `value` 는 아무것도 설명해주지 않는 나쁜 이름의 예시이다. 코드 문맥상 변수가 가리키는 데이터나 값이 아주 명확할 때만 사용하자.
  - 자신만의 규칙이나 소속된 팀의 규칙을 따르기  
     만약 사이트 방문객을 ‘user’라고 부르기로 했다면, 이와 관련된 변수를 `currentVisitor` 나 `newManInTown` 이 아닌 `currentUser` 나 `newUser` 라는 이름으로 통일

변수를 재사용하면 변수 선언에 쏟는 노력을 좀 덜 수는 있겠지만, 디버깅에 더 많은 시간을 써야 한다. **변수를 추가하는 것은 좋은 습관이다!**

### 자료형

자료의 타입은 있지만 변수에 저장되는 값의 타입은 얼마든지 바꿀 수 있으며 이를 **동적 타입(dynamically typed) 언어**라고 부른다.

- **숫자형**
  - 정수 및 부동소수점 숫자
  - **Infinity (무한대)** : 어느 숫자를 0으로 나누거나, 직접 참조한다.
  - **NaN** : 에러 발생을 나타내주는 값으로 예를 들어 문자열을 숫자로 나눴을 때 NaN을 반환하며 연산이 종료된다.
- **BigInt**  
  아주 큰 숫자나 아주 작은 숫자를 필요로 하는 작업(암호 관련 등)에서 쓰이지만 채택된 지 얼마 되지 않은 자료형으로 호환성 이슈가 있다. 숫자 끝에 n을 붙여 선언한다.
  ```javascript
  const bigInt = 1234567890123456789012345678901234567890n;
  ```
- **문자형**  
  큰 따옴표, 작은 따옴표, **역 따옴표(백틱)**을 사용한다.

  ```javascript
  alert("Hello");
  alert("World");
  ```

  큰 따옴표와 작은 따옴표는 기본적인 방법으로 둘의 차이는 없다.

  ```javascript
  let name = "Dougie";

  alert(`Hello ${name}!`); // "Hello Dougie!"
  alert(`Your age is ${4 + 6 + 2}`); // "Your age is 12"
  ```

  역 따옴표(백틱) 사용 시엔 `**${...}**` 안에 변수나 표현식을 넣어주면 문자열 중간에 들어가 있어도 변수/표현식의 값 그대로 문자열의 일부가 될 수 있다. 백틱 사용 시에만 가능하다. (자바나 C에서의 형식지정자로 포맷팅하는 개념과 유사)

- **불린형**
- **null 값**  
  어느 자료형에도 속하지 않는 고유의 값으로, 다른 언어에선 null을 존재하지 않는 객체에 대한 참조나 널 포인터를 나타낼 때 사용하지만 자바스크립트에선 **존재하지 않는 값, 비어 있는 값, 알 수 없는 값**을 나타내는 데 사용한다.
- **undefined 값**  
  null과 같이 고유의 값으로, 변수는 선언했지만 값을 할당하지 않았을 때 해당 변수에 undefined 값이 자동으로 할당된다. 변수에 직접 할당하는 경우는 권유되지 않는다. (null 값 사용)
- **객체형과 심볼형**  
  객체형을 제외한 다른 자료형은 문자열이든 숫자든 한 가지만 표현할 수 있기 때문에 원시(primitive) 자료형이라 부른다. 반면 객체는 데이터 컬렉션이나 복잡한 개체(entity)를 표현할 수 있다.
  심볼형은 객체의 고유한 식별자(unique identifier)를 만들 때 사용된다.

- **typeof 연산**  
  호출하면 인수의 자료형을 나타내는 문자열을 반환한다.
  ```javascript
  typeof undefined; // "undefined"
  typeof 0; // "number"
  typeof 10n; // "bigint"
  typeof false; // "boolean"
  typeof "hi"; // "string"
  typeof Symbol("id"); // "symbol"
  typeof Math; // (1) "object"
  typeof null; // (2) "object"
  typeof alert; // (3) "function"
  ```
  (1) `Math` 는 수학 연산을 제공하는 내장 객체이므로 `"object"` 를 출력한다.  
  (2) 하위 호환성 유지를 위한 언어상의 오류, `null` 은 객체가 아닌 고유한 자료형이다.  
  (3) `"function"` 을 출력하지만 함수형은 따로 없고 함수는 객체형에 속한다. 이 또한 언어상의 오류이다.

### 형변환

- **문자형으로 변환**

  - `alert` : 전달받은 값의 자료형과 관계없이 문자열로 자동 변환해서 출력함
  - `String(value)`

- **숫자형으로 변환**  
  수학과 관련된 함수와 표현식에서 자동으로 일어난다. 숫자형 값을 사용해야 하는데 그 값을 문자 기반 폼(form)을 통해 입력받는 경우엔 명시적 형 변환이 필수적인데, `Number(value)` 함수를 사용한다.

      | 전달받은 값 | 형 변환 후 |
      | --- | --- |
      | undefined | NaN |
      | null | 0 |
      | true / false | 1 / 0 |
      | string | 문자열의 앞뒤 공백 제거 후 남아있는 문자열이 없다면 0 을, 그렇지 않다면 숫자를 읽음. (숫자 이외의 글자가 있는 경우) 변환에 실패하면 NaN 이 됨. |

- **불린형으로 변환**  
  `Boolean(value)`
  - 숫자 0, 빈 문자열, `null`, `undefined`, `NaN` 과 같이 직관적으로 비어있다고 느껴지는 값들은 모두 `false`
  - 그 외의 값은 `true`
  - 비어있지 않은 문자열 `**“0”**`, `**“ “**`(공백)의 경우에도 `**true**`

### 기본 연산자와 수학

- **주요 용어**
  - 피연산자(operand) : 연산자가 연산을 수행하는 대상으로 인수(argument)라고도 불린다.
  - 단항(unary) 연산자 : 피연산자를 하나만 받는 연산자. 대표적으로 `-` 연산자, 부호를 뒤집는다.
  - 이항(binary) 연산자 : 두 개의 피연산자를 받는 연산자
- **연산자 종류**

  - 덧셈 연산자 `+`
  - 뺄셈 연산자 `-`
  - 곱셈 연산자 `*`
  - 나눗셈 연산자 `/`
  - **나머지 연산자 `%`**
  - **거듭제곱 연산자 `**`**  
`a \*\* b`를 평가하면`a`를`b`번 곱한 값이 반환된다. 아래와 같이 제곱근을 구할 수 있다.
    `javascript
    alert(4 ** (1 / 2)); // 2 (1/2 거듭제곱은 제곱근)
    alert(8 ** (1 / 3)); // 2 (1/3 거듭제곱은 세제곱근)
    `

- **이항 연산자 `+` 와 문자열 연결**

  덧셈 연산자 `+` 는 보통 숫자로 된 값을 반환하지만, **피연산자 중 하나라도 문자열이면 문자열로 병합되어 반환**된다. 다른 연산자에는 적용되지 않는다.

  ```javascript
  alert("this is " + 6); // "this is 6"
  alert(2 + "1"); // "21"

  alert(2 + 1 + "1"); // "31"
  alert("2" + 1 + 1); // "211"

  alert("2" - 1); // 1, '2'를 숫자로 바꾼 뒤 연산됨
  alert("6" / "2"); // 3, 두 피연산자를 숫자로 바꾼 뒤 연산됨
  ```

- **단항 연산자 `+` 로 숫자형으로 변환**

  단항 연산자 `+` 를 붙이면 숫자엔 아무런 영향을 미치지 않지만, 숫자형이 아닌 피연산자는 숫자형으로 변환된다.

  ```javascript
  let x = -1;
  alert(+x); // -1

  alert(+true); // 1
  alert(+""); // 0

  let apples = "2";
  let oranges = "3";
  // 이항 덧셈 연산자가 적용되기 전에 두 피연산자는 숫자형으로 변환됨
  alert(+apples + +oranges); // 5

  // Number(...)로 작성할 수 있지만 더 길다
  alert(Number(apples) + Number(oranges)); //5
  ```

- **연산자 우선순위**

  하나의 표현식에 둘 이상의 연산자가 있는 경우, 실행 순서는 연산자의 우선순위에 의해 결정된다. 순위가 같으면 왼쪽에서 오른쪽으로 연산되고, 괄호는 모든 연산자보다 우선순위에 있기 때문에 복잡한 표현식의 경우 명시적으로 괄호를 써 주는 것이 좋다.

- **할당 연산자**  
  무언가를 할당할 때 쓰는 연산자 `=` 이다. 우선순위는 아주 낮아서, 표현식에서 계산이 이루어지고 값이 할당되는 이유가 바로 이 때문이다.
- **복합 할당 연산자**

  ```javascript
  let n = 0;
  n += 4; // n = n + 4
  n /= 2; // n = n / 2
  ```

  변수에 연산자를 적용하고 그 결과를 같은 변수에 저장해야 할 때 쓰는 짧은 문법으로, 할당 연산자와 우선순위가 같다. `+=`, `-=`, `/=`, `*=`, `%=`, `**=` 를 쓸 수 있다.

- **증가/감소 연산자**

  변수의 크기를 하나 늘리거나 줄이는 짧은 문법.

  - 후위형(postfix form) : `count++`, 증가/감소 전의 기존 값을 반환한다.
  - 전위형(prefix form) : `++count`, 증가/감소 후의 새로운 값을 반환한다.

- **비트 연산자**

  비트 연산자(bitwise operator)는 인수를 32비트 정수로 변환하여 이진 연산을 수행한다.

  - 비트 AND `&`
  - 비트 OR `|`
  - 비트 XOR `^`
  - 비트 NOT `~`
  - 왼쪽 시프트(LEFT SHIFT) `<<`
  - 오른쪽 시프트(RIGHT SHIFT) `>>`
  - 부호 없는 오른쪽 시프(ZERO-FILL RIGHT SHIFT) `>>>`

- **쉼표 연산자 `,`**  
  여러 표현식을 코드 한 줄에서 평가할 수 있게 해준다. 각각 모두 평가되지만 마지막 표현식의 결과만 반환된다. 쉼표 연산자의 우선순위는 할당 연산자보다도 낮기 때문에 괄호를 치지 않으면 맨 앞의 값이 할당되고 끝난다.

  ```javascript
  a = (1 + 2, 3 + 4);
  alert(a); // 7

  let a = 1 + 2, 3 + 4;
  alert(a); // 3
  ```

### 비교 연산자

- **비교 연산자의 종류**
  - 보다 큼/작음 : `a > b` / `a < b`
  - 보다 크거나/작거나 같음 : `a >= b` / `a <= b`
  - 같음 : `==`
  - 같지 않음 : `!=`
- **비교 연산자는 불린형을 반환**하며, 변수에 할당할 수 있다.

  ```javascript
  let a = 4 > 5; // 결과를 a 에 할당
  alert(a); // false
  ```

- **문자열 비교**

  자바스크립트는 사전편집(lexicographical)순으로 문자열을 비교한다. 사전 뒤쪽으로 갈수록 크다고 판단한다. 정확히는 유니코드순이므로 대문자보다 소문자가 더 크다. 글자를 하나하나 비교하며, 비교가 종료되었지만 두 문자열의 길이가 다르다면 길이가 긴 문자열이 더 크다고 판단한다.

  ```javascript
  alert("Z" > "A"); // true
  alert("Glow" > "Glee"); // true
  alert("Bee" > "Be"); // true
  ```

- **다른 형을 가진 값 사이의 비교**

  비교하려는 값의 자료형이 서로 다르면 자바스크립트는 이 값을 모두 숫자형으로 바꾼다.

  ```javascript
  let a = 0;
  alert(Boolean(a)); // 숫자 0, false

  let b = "0";
  alert(Boolean(b)); // 빈 문자열이 아닌 문자열, true

  alert(a == b); // 문자열 "0"을 숫자형로 변환하여 true!
  ```

- **일치 연산자**

  `false` 는 숫자형으로 변환하면 `0` 이기 때문에, 동등 연산자 `==` 로 `0` 과 `false` 를 구분할 수가 없다. 빈 문자열 `''` 도 마찬가지다. 이럴 땐 일치 연산자(strict equality operator) `===` 를 사용하면 된다. 값 뿐만 아니라 자료형의 동등 여부까지 검사한다. `!==` 도 부등 연산자의 엄격한 버전이다.

- **`null` 이나 `undefined` 와 비교하기**
  - `null` 과 `undefined`
    ```javascript
    alert(null == undefined); // true
    alert(null === undefined); // 자료형이 다르기 때문에 false
    ```
  - `null` 과 `0`
  ```javascript
  alert(null > 0); // null을 숫자형으로 변환했으므로 false (0 > 0)
  alert(null >= 0); // true (0 >= 0)
  alert(null == 0); // null은 undefined를 만날때만 true이므로 false
  ```
  - **비교가 불가능한 `undefined`**  
    `undefined` 은 비교시 `NaN` 으로 변환되는데 `NaN` 이 피연산자인 경우 비교 연산자는 항상 `false` 를 반환한다. `undefined` 는 `null` 이나 `undefined` 와 같고, 이외의 값과는 같지 않다.

### 논리 연산자

- **논리 연산자의 종류**

  - OR 연산자 `||` : 하나만 `true` 여도 `true` 반환
  - AND 연산자 `&&` : 인수가 모두 `true` 여야만 `true` 반환
  - NOT 연산자 `!` : 피연산자를 불린형으로 변환하고 그 역을 반환한다.
    NOT 연산자를 두 개 연달아 사용하면 값을 불린형으로 바꿀 수 있다. (역의 역)
    ```javascript
    alert(!true); // false
    alert(!!null); // false
    alert(!!"hello"); // true
    ```

- **첫 번째 truthy를 찾는 OR 연산자 `||`**  
  OR 연산자를 체이닝한 경우, **1.** 가장 왼쪽부터 시작해 피연산자를 평가한다. **2.** 각 피연산자를 불린형으로 변환후 그 값이 `true` 이면 해당 피연산자의 변환 전 값을 반환한다. **3.** 모두 `false` 이면 마지막 피연산자를 반환한다. 이런 기능을 통해 여러 용도로 OR 연산자를 사용할 수 있다.

  1. **변수 또는 표현식으로 구성된 목록에서 첫 번째 truthy 얻기**

     ```javascript
     let firstName = "";
     let lastName = "";
     let nickName = "바이올렛";

     alert(firstName || lastName || nickName || "익명");
     // "바이올렛", 모든 변수가 falsy이면 "익명" 출력
     ```

  2. **단락 평가(short circuit evaluation)**

     OR 연산자는 왼쪽부터 시작해 오른쪽으로 평가를 진행하는데, truthy를 만나면 나머지 값들은 건드리지 않은 채 평가를 멈춘다. 이런 프로세스를 단락 평가라고 한다.

     ```javascript
     true || alert("not printed"); // 얼럿 실행되지 않음
     false || alert("printed"); // 얼럿 실행 됨
     // 왼쪽 조건이 falsy일 때만 명령어를 실행하고자 할 때 많이 쓴다.
     ```

- **첫 번째 falsy를 찾는 AND 연산자 `&&`**  
  AND 연산자를 체이닝한 경우, OR 연산자 알고리즘과 유사하다. **1.** 가장 왼쪽부터 시작해 피연산자를 평가한다. **2.** 각 피연산자를 불린형으로 변환후 그 값이 `false` 이면 해당 피연산자의 변환 전 값을 반환한다. **3.** 모두 `true` 이면 마지막 피연산자를 반환한다.
  `&&` 연산자의 우선순위는 `OR` 연산자보다 높다.

### if문

- `**if(condition) { statement; }`** 조건이 true일 때만 문을 실행한다. 자바와 동일하게 `**if-else`, `if-else if-else`, `nested if`\*\* 작성 가능하다.

- **조건부 연산자 `?` (삼항 연산자)** : 피연산자를 세 개 받는다.

  ```javascript
  let accessAllowed = age > 18 ? "Bye" : "Hi";
  // 괄호 안의 조건이 true이면 "Bye"를 할당, false이면 "Hi"를 할당
  ```

  물음표 연산자를 다중으로 사용하는 경우도 있다. 보다시피 `if-else if-else` 와 유사한 구조이다. 물음표 연산자를 `if` 대용으로 쓰는 경우가 종종 있는데, 물음표 연산자 `?` 는 조건에 따라 반환 값을 달리하려는 목적으로 만들어졌기 때문에 여러 분기를 만들어 처리할 때는 `if` 를 사용하는것이 좋다.

  ```javascript
  let age = prompt("나이를 입력해주세요.", 18);
  // prompt(message, default);

  let message =
    age < 3
      ? "아기야 안녕?"
      : age < 18
      ? "안녕!"
      : age < 100
      ? "환영합니다!"
      : "나이가 아주 많으시거나, 나이가 아닌 값을 입력 하셨군요!";

  alert(message);
  ```

### switch문

- **복수의 if문은 switch문으로** 바꿀 수 있다.

  ```javascript
  let a = 2 + 2;

  switch (a) {
    case 2:
      alert("비교하려는 값보다 작습니다.");
      break;
    case 3:
      alert("비교하려는 값보다 작습니다.");
      break;
    case 4:
      alert("비교하려는 값과 일치합니다.");
      break;
    case 5:
      alert("비교하려는 값보다 큽니다.");
      break;
    case 6:
      alert("비교하려는 값보다 큽니다.");
      break;
    default:
      alert("어떤 값인지 파악이 되지 않습니다.");
  }
  ```

- 스위치문은 `break` 문을 만나거나 `switch` 문이 끝나면 코드의 실행을 멈추는데, 이와 같은 특징을 역이용하여 **여러 개의 case문을 묶을 수 있다. (fall through)**

  ```javascript
  let a = 2 + 2;

  switch (a) {
    case 2: // 2인 경우 이 case 실행, 그러나 break이 없기 때문에 다음 case로 넘어감
    case 3: // 3인 경우에도 얼럿 실행 후 break으로 switch문 종료
      alert("비교하려는 값보다 작습니다.");
      break;
    case 4:
      alert("비교하려는 값과 일치합니다.");
      break;
    case 5: // 5인 경우 이 case 실행, 그러나 break이 없기 때문에 다음 case로 넘어감
    case 6: // 5인 경우에도 얼럿 실행 후 break으로 switch문 종료
      alert("비교하려는 값보다 큽니다.");
      break;
    default:
      alert("어떤 값인지 파악이 되지 않습니다.");
  }
  ```

- switch문은 일치 비교 `===` 로 조건을 확인한다. **형과 값이 모두 같아야 해당 case문이 실행**됨을 유의.
