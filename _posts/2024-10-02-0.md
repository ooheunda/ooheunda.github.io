## 프로젝트 회고

### social-feed-hub

- 게시물 목록 조회 API를 구현하며 잘 모르고 그냥 사용했던 JOIN 이라는 개념(inner, left…)을 제대로 공부하고 적용해볼 수 있어서 좋았다.
- 목록 조회 API에서 쿼리 빌더로 데이터를 가져오는 부분을 private 함수로 뺀다던지, (이건 팀 내에서 논의했던 부분이라 보편적으로 적용이 안되긴 하지만) ORM API로 가능한 부분은 최대한 그렇게 쓴다던지 등 어떻게 재사용성이 높고 가독성이 좋은 코드를 짤 수 있는지 코드 스타일에 대해 생각해보게 되었다.
- 개인 느낀점) 두 달 넘게 개발을 안하다가 굉장히 구체적으로 컨벤션이나 DB 모델, 명세서 등을 정해놓고 진행했던 프로젝트라 좋았다. 코드리뷰도 활발해서 좋았음.

### matjum

- ESLint, github issue, PR 템플릿 등 개발 환경, 컨벤션을 넓고 구체적으로 정할 수 있다는 걸 알게 되어서 약간 신세계였고 너무 좋았다.
- 아무것도 모르고 `docker-compose up` 만 했던 과거를 청산하고 도커와 컨테이너 개념에 대해 조금이나마 알게 되어서 좋았다. 직접 설정한 건 아니지만 열심히 설명해주셔서 너무 좋았음
- 테스트 코드 중요하다 얘기만 듣다가 직접 작성해본건 거의 처음(express에서만 한 번 써봄… 아주 오래전)이었는데 테스트 코드를 작성하며 버그를 발견하는등 정말 중요성을 깨닫게 되어 좋았다. [**(관련 PR)**](https://github.com/INgandle/matjum/pull/32) 그리고 처음에 어떻게 써야할지 막막했는데 지수님이 GPT 팁을 주셔서 진짜 그거 아니었으면 아직도 안 쓰고 있었을 듯…
- 배포를 맡았었는데 EC2, RDS는 평소와 비슷하게 사용했지만 IAM에 대해 생각해본건 처음이라서 좋았다. 원래 루트 계정은 잘 사용하지 않는다는 사실과 권한 관리에 대해 생각해볼 수 있었다.
- 디스코드 웹훅에 대해서 알게 되어서 좋았다.

### 기업과제 1

- MSA 얘기만 듣다가 NestJS에서 모노레포 모드를 구성해보며 약간 쁘띠 MSA를 경험해본 것 같아서 이런 것도 있구나 알게 되어서 좋았다. 그런데 공통 파일(`libs/`)에 대해서는 이해가 잘 안되어서 import 하는 것이나 어떻게 구조를 짜야 하는지 생각하는게 힘들었다.
- RPC라는 기술이 있다라는 것을 알게 되어서 좋았다.

## 전체 회고

- 오랜만에 팀 프로젝트를 하면서 부트캠프때 했던 컨벤션 관리는 정말 빙산의 일각이었구나를 알게 되었다. 커밋, Issue, PR, lint, prettier, 용어 정리, 코드 스타일, 리뷰 룰 등 세세하게 정할 수록 작업할 때엔 수월해진다는 점을 체감하게 되어서 좋았고 앞으로도 열심히 적용할 것 같다.
- 매일 스크럼에서 고민사항이나 상황 공유 등을 하면서 내 업무가 아니어도 어떻게 기능하는지, 다른 팀원은 어떤 에러를 어떻게 해결하는지 지식 도둑질을 할 수 있어서 좋았다. (코드 리뷰는 안하면 10손해라는 어떤 개발자의 말씀을 정말 알게 되엇음…)
- 팀 프로젝트 하는 건 너무 좋았는데 정규 수업에 집중을 잘 못해서 개인적으로 아쉬웠고 (내용 자체도 좀… 아쉬웟.음.) 개인 프로젝트 기간엔 아예 한량처럼 놀아서 지나고보니 후회스럽다. 하지만 이번주까진 놀것이다.
- 문서화의 중요성을 깨닫게 되었다. 기록을 남기지 않으면 내 머릿속엔 아무것도 남지 않는다…
